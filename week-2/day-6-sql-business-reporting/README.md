# Day 6 - SQL Business Reporting Sprint

## Practice Goal

The goal of this practice was to learn how to use SQL for business reporting and transform raw data into useful business insights. I practiced writing queries using aggregations, GROUP BY reports, HAVING filters, and JOINs to connect multiple tables and create meaningful reports for managers.

## Files Included

- `setup.sql` - Creates the database tables and inserts the customers, products, and orders data.
- `basic_aggregations.sql` - Contains basic business metrics using COUNT, SUM, AVG, MIN, and MAX.
- `group_by_reports.sql` - Creates grouped reports using GROUP BY and HAVING.
- `join_reports.sql` - Uses JOINs to combine tables and create detailed business reports.
- `business_report.md` - Contains business insights based on SQL query results.
- `query_explanations.md` - Explains important SQL queries and their business meaning.

## How to Run SQL Files

Run the SQL files in this order:

1. Run `setup.sql`
   - Creates the tables.
   - Inserts all sample data.

2. Run `basic_aggregations.sql`
   - Calculates basic metrics such as total orders, completed orders, and revenue.

3. Run `group_by_reports.sql`
   - Creates grouped reports by status, date, customer, and product.

4. Run `join_reports.sql`
   - Combines tables using JOINs to create detailed business reports.

5. Review:
   - `business_report.md`
   - `query_explanations.md`

## SQL Concepts Practiced

### Basic Aggregation

Basic aggregation is used when we need one summary result from data. Functions like COUNT, SUM, AVG, MIN, and MAX help answer questions such as total orders, average prices, or total revenue.

### GROUP BY

GROUP BY is used when we want to split data into categories and create a report for each group. For example, counting orders by status or calculating revenue by product.

### HAVING

HAVING is used after GROUP BY to filter grouped results. For example, showing only customers who placed more than one order or products with completed quantity greater than two.

### JOIN

JOIN is used to combine information from different tables. For example, the orders table contains product_id and customer_id, but the product names, prices, and customer cities are stored in other tables. JOIN allows us to create complete reports with meaningful information instead of only showing IDs.

## Most Important Business Insight

The most important insight found from the reports was that completed revenue was mainly generated by electronics products, especially laptops and monitors. The reports also showed that completed orders should be separated from pending and cancelled orders because they do not represent real revenue.

This type of reporting helps managers understand sales performance, identify valuable products and customers, and make better business decisions.
